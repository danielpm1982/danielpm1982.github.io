Creating an AI Agent with Google Gemini LLM
(Python, LangChain, LangGraph, RAG)

Hello guys !! I've just finished implementing this Gemini AI Agent, customized with Retrieval-Augmented Generation (RAG) training, and built with LangChain and LangGraph. All that coded in Python. This Generative AI Agent would manage a sample scenario of Client Support for a company site. For now, there are only some few sample policies defined, at three sample pdf files, which could be updated later, or be added with additional local or remote further data/information, in order to increase the "knowledge" of the model and its ability to respond to more questions. The model was set to reason only based on the custom information passed through the policies' pdf files, and not on previous Google general data trainings - this helps avoid AI hallucinations when working with specific local tasks. In this project, we've used Google Gemini, Google Colab (Jupyter Notebook), Python and the following specific Python libs/modules:

- google.colab
- langchain_google_genai
- pydantic
- typing
- langchain_core.messages
- pathlib
- langchain_community.document_loaders
- langchain.text_splitter
- langchain_community.vectorstores
- langchain_core.prompts
- langchain.chains.combine_documents
- langgraph.graph
- IPython.display

For you to RUN the code, you should:

- open this project link at your Google Colab account, after logging in;
- have your own valid Google Api Key for accessing Google AI services and add it to the secrets section of your Colab as GOOGLE_API_KEY. This environment variable is the one called at the application, when Gemini is to be used;
- upload the three sample policies' pdfs from the following link to your Colab account as well, so that this app can access them. The pdfs are available for download at: Project Policies' PDFs repository .
